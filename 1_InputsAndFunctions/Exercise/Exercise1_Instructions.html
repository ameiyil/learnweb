Exercise 1 Instructions

In this exercise, you'll but completing a web page that will convert a temperature from Fahrenheit to both Celsius and Kelvin.
Then the results will be displayed in the text area on the page.

Go to the "Answers" folder and double click on answer.html to open it in Chrome to see what the result will look like.

When you're ready, follow along with the steps.

Each step on this page that relates to changing some code will have a corresponding "TODO: Step #_" in the exercise.html, equations.js, or styles.css pages.
Look for that text and fill in your answer.

Getting Started:

  • Download Visual Studio Code
    Go to https://code.visualstudio.com/download and click on the "Windows" download button, then install the program
  • Open Visual Studio Code
    Open the Visual Studio Code program from the shortcut it created on your desktop
  • Open the exercise
    Go to File > Open Folder ... > Select the "Exercise" folder that these instructions are in
  • Open the exercise.html in Visual Studio Code
    On the left, click on the paper icons and double click on "exercise.html" in the left menu that gets opened
  • Open up the exercise page in Chrome
    Now we're going to open it in Chrome to see that the web page actually looks like. Double click on exercise.html to open it in Chrome. Since it has the .html extension, the computer knows that it can be opened in a browser.
  • Give a name to the web page
    Right now this page doesn't have a name, look for the <title> tags and put the title you want between them.

Creating the structure of the page:

  1. Give a name to the web page
    • Right now this page doesn't have a name, look for "TODO: Step #1" and the <title> tags and put the title you want between them.
    • Save your changes and then refresh the exercise page in Chrome and notice that the name in the tab has changed
  2. Add a label for the text field
    • Right now the page only has a button and a text area for the results. But there isn't anywhere for anyone to type in the temperature
    • Look for "Step #2" in exercise.html
    • The label element is the text that is shown next to the text box
    • Type some text between the <label> tags
    • Save your changes and then refresh the exercise page in Chrome
  3. Add a text field
    • Now that we have a label for the text field we need to add the box for people to type stuff in
    • Right after where you typed the name for the label and also between the <label> tags, type <input id="???" type="text" value="???"></input>
    • For the id attribute, name it something meaningful like "fahrenheit" - we will use this is to access it's value later on
    • For the value attribute, put what you think the default value should be, for example 0 or 32
    • Save your changes and then refresh the exercise page in Chrome

Adding the look of the page:

  1. Add some styling
    • Without some styling, the page looks really ugly right now
    • We already have styles defined in styles.css, we just need to tell the exercises page to use them
    • To do this, add the line <link rel="stylesheet" type="text/css" href="styles.css"> anywhere between the head tags
    • Save your changes and then refresh the exercise page in Chrome
    • It should look a bit cleaner now

Adding the behaviour of the page:

  1. Add behaviour when you click the calculate button
    • Right now, clicking on the button doesn't do anything, even though the onclick attribute has a value "calculate()"
    • This is because the page currently doesn't know where to look for that JavaScript function
    • We will tell it where to find this code by adding the line <script src="equations.js"></script> in between the head tags
    • Save your changes and then refresh the exercise page in Chrome
    • Now click on the button again, you'll see that stuff happens but not what we want, we'll fix that in the next step
  2. Tell the code to get the value we put in the text field that we added in Step 3
    • Open the equations.js file
    • Create a variable called "fahrenheit" using the "let" keyword, this creates a variable that can be assigned different values later on: let fahrenheit = document.getElementById('???').value;
    • Where I've put ???, type in the id that you chose in step 3
  3. Convert the value of the "fahrenheit" variable from a "String" to a "Number"
    • The value that is currently stored in our fahrenheit variable is currently a String
    • In order for us to perform mathematical calculations on it, we must first convert it into a Number
    • To do this, re-assign the value of fahrenheit to its value converted into a Number using this code fahrenheit = parseFloat(fahrenheit)
    • This might seem strange that we are assigning a variable but also performing calculations on it at the same time
    • What the computer is doing is first calculating what's on the right of the "=" sign, and then putting that calculated value into the variable on the left of the "=" sign
  4. Create a variable to store the temperature in Celsius
    • This time we will create a variable called "celsius" using the "const" keyword, this creates a variable whose value can never change later on
    • The benefit of this is that it allows us to know that once it is assigned a value, it will never by anything other than that
    • Type in the following code: const celsius = fahrenheitToCelsius(fahrenheit);
  5. Update our output variable with the Celsius value we just calculated
    • output = output + fahrenheit + "&deg;F is " + celsius + "&deg;C";
    • What we are doing above is building a string by adding up all the parts
    • By doing output = output + ..., this tells the computer to add our new strings on top of what we already have in the output variable
    • When variables fahrenheit and celsius are added to other strings, the computer converts its values to a string first
    • This process allows us to give output the value of a string, but values stored those variables are untouched and remain as numbers
  6. Get the text area and assign its value to our output variable's value
    • Get the text area element by calling the document.getElementById function with its id "results"
    • Assign it to a new variable called "textArea": const textArea = document.getElementById('results');
    • Now using that textArea variable, we can set its text to our output: textArea.innerHTML = output;
    • Save your changes and then refresh the exercise page in Chrome
    • Click on the button and notice that now we are actually seeing a value in the text area
    • Try it out a few times with different fahrenheit values, what do you notice?
  7. Fix the fahrenheitToCelsius function
    • You'll have noticed that not matter what fahrenheit value you've put, the calculated celsius value is always 0
    • Take a look at what is inside the fahrenheitToCelsius function, it's always returning 0
    • Change what is returned with the actual formula
    • Save your changes and refresh the page in Chrome to test if your function is correct
    • Keep updating your code until it is
  8. Output the Fahrenheit value converted to Kelvin as well
    • Repeat step 8 to create a variable called "kelvin"
    • Repeat step 9 to update our output with the new kelvin value we just added
    • Save your changes and then refresh the exercise page in Chrome
    • Click on the button, what do you see?
  9. Put the Kelvin temperature on the next line
    • You may have noticed that the Celsius and Kelvin lines are squished together
    • To tell the computer to put the Kelvin value on the next line, we add a special string "\n" to where we want the line break to be
    • You'll want something looking like this: output = output + "\n" + ...
    • Save your changes and then refresh the exercise page in Chrome
    • Try it out a few times with different fahrenheit values, what do you notice?
  10. Fix the celsiusToKelvin
    • Like before you'll notice that the Kelvin value is also always 0
    • Fix the celsiusToKelvin function to return the correct values
    • Save your changes and refresh the page in Chrome to test if your function is correct
    • Keep updating your code until it is and then you're done!
  11. Bonus: Change the number of significant digits
    • You might have noticed that the calculated values sometimes have a bunch of significant digits
    • We're only interested in up to 2 decimal points
    • We can change this by modifying the temperature values as they are being stored in the output variable
    • To do this we add a function call .toFixed(2) to each of the temperature values
    • Save your changes and then refresh the exercise page in Chrome
    • You should now notice that something like 17.11111111 becomes 17.11

Congrats! You are now done!
